## 2. Risk Assessment and Mitigations

### 2.1 High Priority Risks

| Risk | Likelihood | Impact | Mitigation | Owner |
|------|------------|--------|------------|-------|
| fluidlite FFI crashes on some platforms | Medium | High | Feature-gated; fallback to MP3; comprehensive error handling | Dev |
| Soundfont file corruption causes panic | Low | High | Panics caught; state set to Fallback; continue with MP3 | Dev |
| Cross-platform path resolution failures | Medium | High | Extensive testing on all platforms; multiple fallback paths | Dev |

### 2.2 Medium Priority Risks

| Risk | Likelihood | Impact | Mitigation | Owner |
|------|------------|--------|------------|-------|
| Soundfont memory bloat on low-end devices | Medium | Medium | Lazy loading; voice limits; disable option | Dev |
| Audio glitches on fallback transition | Low | Low | Maintain both paths; seamless switch | Dev |
| Build failures on some platforms (no C toolchain) | Medium | Medium | Optional feature; clear error messages; CI matrix | Dev |

### 2.3 Low Priority Risks

| Risk | Likelihood | Impact | Mitigation | Owner |
|------|------------|--------|------------|-------|
| MIDI timing issues with high polyphony | Low | Medium | Test with various polyphony levels; adjust voice limit | Dev |

### 2.4 Risk Mitigation Strategy

1. **Feature Gating**: Soundfont is optional (`--features soundfont`)
2. **Automatic Fallback**: MP3 playback when soundfont fails
3. **Comprehensive Logging**: All errors logged for debugging
4. **Incremental Adoption**: Existing behavior unchanged by default
5. **Cross-Platform Testing**: CI runs on Windows, macOS, Linux

## Soundfont Synthesis Support (New Feature)

### Features
- Software synthesizer playback using SoundFont (.sf2) files
- Lower latency audio compared to MP3 samples
- Automatic fallback to MP3 if soundfont fails to load

### Usage
```bash
# Use default soundfont
./pt2 --file level.json

# Use custom soundfont
./pt2 --file level.json --soundfont /path/to/piano.sf2

# Disable soundfont (MP3 only)
./pt2 --file level.json --no-soundfont
```

### System Requirements
- Windows: Visual C++ Redistributable (for fluidlite)
- macOS: CoreAudio
- Linux: ALSA or PulseAudio
```

### 6.2 SoundfontManager Module

```rust
// === Public API ===

/// Create a new synthesizer instance
pub fn create_synthesizer() -> SoundfontResult<Box<dyn SoundfontSynth>>

/// Resolve soundfont path with cross-platform support
pub fn resolve_soundfont_path(
    cli_path: Option<&str>,
    exe_dir: &Path,
) -> (PathBuf, &'static str)

/// Validate soundfont path
pub fn validate_soundfont_path(path: &str) -> SoundfontResult<PathBuf>

// === Types ===

pub enum SoundfontState {
    Uninitialized,
    Loading,
    Ready,
    Fallback,
    Disabled,
    Unloading,
}

pub enum SoundfontError {
    FileNotFound(String),
    FileReadError(String),
    ParseError(String),
    SynthInitError(String),
    NoPresets(String),
    NotEnabled,
    AlreadyLoaded,
    NotLoaded,
}

pub struct SoundfontConfig {
    pub path: String,
    pub sample_rate: u32,
    pub voices: u32,
    pub reverb: bool,
    pub chorus: bool,
    pub volume: f32,
    pub channel: u8,
}

pub trait SoundfontSynth: Send + Sync {
    fn note_on(&self, channel: u8, note: u8, velocity: u8);
    fn note_off(&self, channel: u8, note: u8);
    fn all_notes_off(&self);
    fn is_ready(&self) -> bool;
    fn state(&self) -> SoundfontState;
    fn config(&self) -> Option<SoundfontConfig>;
    fn reset(&self);
}
```
